# ChkDsk FAT32 — плагин проверки и ремонта диска для Wild Commander (TS-Conf)

Плагин для файлового менеджера **Wild Commander** (платформа **TS-Conf** / Pentevo,
эмулятор Unreal Speccy). Проверяет и чинит файловую систему **FAT32** на текущем
смонтированном диске (SD-карта через Z-Controller): отчёт MS-DOS-стиля + детект и
ремонт типовых ошибок FAT.

> Подробный учебник по дисковой подсистеме TS-Conf, FAT32 и написанию плагинов WC —
> в [`docs/TS-Conf_дисковая_подсистема_и_плагины_WC.md`](docs/TS-Conf_дисковая_подсистема_и_плагины_WC.md).

---

## Что это и зачем

Аналог `chkdsk`/`fsck` для FAT32, но как плагин Wild Commander. Работает напрямую с
SD-карту через хардварный SPI-контроллер (Z-Controller), читает BPB и FAT, обходит
дерево каталогов и приводит файловую систему в согласованное состояние.

Главное окно показывает отчёт о томе (как `CHKDSK` в MS-DOS):
```
                 Volume report

  103201792 bytes total disk space
      34304 bytes in    18 directories
   59819748 bytes in   535 user files
   43227136 bytes available on disk

        512 bytes in each allocation unit
     201566 total allocation units
      84428 available allocation units

         17 lost allocation units
          0 backup-FAT mismatches
```

## Что умеет (детект и ремонт)

| Ошибка | Детект | Ремонт | Класс |
|--------|--------|--------|-------|
| **Потерянные кластеры** (lost) — заняты в FAT, но недостижимы | да | обнулить записи FAT (освободить место) | детерминированный |
| **Рассинхрон копий FAT** (backup-FAT mismatch) | да (посекторно) | скопировать хорошую копию поверх плохой | детерминированный |
| **Битый заголовок FAT** (`FAT[0]` media / `FAT[1]` EOC) | да (даже когда обе копии битые одинаково) | из хорошей копии, либо реконструкция из BPB | детерминированный |
| **Несовпадение размера и цепочки** (bad-size) | да (deep check) | размер в каталоге ← фактическая длина цепочки | детерминированный |
| **Перекрёстные ссылки** (cross-link) | да (deep check) | обрезать второго претендента (best-effort) | с возможной потерей хвоста |

- Поддержка как блочной (SDHC/SDXC), так и байтовой адресации SD.
- Корректно работает с media-байтом **`0xF0`** (removable) — типичным для SD/образов.
- Прогресс-бар (псевдографика) и проценты во время длительных операций.
- Оконный bitmap достижимости (16 КБ на странице `#C000`) → ремонт укладывается в 2
  прохода на ~200k-кластерном томе.

## Что НЕ умеет (ограничения)

- **Не спасает данные потерянных цепочек в файлы** (нет аналога `FOUND.NNN`/`*.CHK`) —
  потерянные кластеры просто возвращаются в свободное место.
- **Cross-link чинится только best-effort**: владелец (первый по обходу) сохраняет
  данные, второй файл теряет общий хвост (структурно FAT становится консистентной, но
  целостность данных второго файла не гарантируется — это в принципе невозможно).
- **bad-size**: безопасный вариант (размер ← цепочка); не пытается «достроить»
  недостающие кластеры.
- Не дефрагментирует, не меняет геометрию, не создаёт/не удаляет разделы.
- Рассчитан на **FAT32** (не FAT12/16, не exFAT/NTFS).
- Номера функций API и поведение соответствуют целевой версии WC (формат заголовка
  `#0A`, WC v1.01); на других сборках WC сверяйтесь с `src/wc_api.h.a80`.

## Установка

1. Возьмите готовый плагин [`CHKDSK.WMF`](CHKDSK.WMF) (или соберите из исходников, см. ниже).
2. Скопируйте `CHKDSK.WMF` в каталог `\WC\` на диске Wild Commander.
3. Зарегистрируйте плагин в `\WC\wc.ini`, в секции `[PLUGINS]`, добавив строку:
   ```
   CHKDSK.WMF
   ```

Для образа `wc.img` под эмулятором всё это делает скрипт-инжектор автоматически:
```
python inject_chkdsk_to_wc_img.py --img path\to\wc.img
```
(копирует `src/CHKDSK.WMF` в `\WC\` и правит `wc.ini`).

## Использование

1. Запустите Wild Commander, откройте меню плагинов и выберите **ChkDsk FAT32**.
2. Плагин просканирует текущий диск и покажет отчёт о томе.
3. Управление (показывается в нижней строке окна):
   - **SPACE = deep check** — глубокая проверка цепочек: ищет cross-link и bad-size
     (медленнее: полный обход всех цепочек, с прогресс-баром).
   - **ENTER = repair** — починить найденное (lost / зеркало FAT / заголовок /
     bad-size / cross-link). Появляется, только когда есть что чинить.
   - **ESC = exit** — выход.

Порядок ремонта (как в dosfstools): сначала копии/заголовок FAT, затем потерянные
кластеры, затем bad-size, затем cross-link.

> ⚠️ Ремонт изменяет диск. Cross-link-ремонт может усечь файл. Для важных данных
> сделайте резервную копию образа перед запуском ремонта.

## Сборка из исходников

Нужен ассемблер **sjasmplus**.
```
cd src
sjasmplus --sym=dbg.sym main.a80      # -> CHKDSK.WMF
```
Или одной командой (сборка + инжект в образ) — `build.bat [path\to\wc.img]`.

Исходники (`src/`): `main.a80` (точка входа + переходники API), `chkdsk.a80` (основная
логика), `bitmap.a80` (bitmap достижимости + обход цепочек), `fmt.a80` (32-бит
арифметика), `sd_zc.a80` (драйвер SD/Z-Controller), `wc_api.h.a80` / `wc_header.a80` /
`tsconf.h.a80` / `colors.h.a80` (заголовки). Все комментарии — на русском (UTF-8).

## Тестирование без железа

- `sim_chkdsk.py` — детерминированный Z80-харнесс: грузит `CHKDSK.WMF`, мокает API WC,
  эмулирует Z-Controller поверх FAT32-образа и прогоняет процедуры (детект/ремонт) с
  лимитом инструкций. Требует чистого Python-ядра Z80 от cburbridge в каталоге
  `_z80_lib_cburbridge/` (не входит в репозиторий, см. ссылку в «Источниках»).
- `make_test_img.py` — сгенерировать маленький тестовый FAT32-образ.
- Скрипты внесения ошибок (с откатом `--restore`, где применимо) для проверки ремонта:
  `make_lost_clusters.py`, `make_fat_mismatch.py`, `make_fat0_corrupt.py`,
  `make_fat_both_bad.py`, `make_deep_errors.py`.

## Структура репозитория

```

CHKDSK.WMF      готовый собранный плагин

```
